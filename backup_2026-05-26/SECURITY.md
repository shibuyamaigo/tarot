# セキュリティ・チート対策

> 「Zwiftで実際に起きているチート問題」を未然に防ぐ。
> ユーザーの信頼こそがリーダーボードの価値を生む。

---

## 🛡 4層防御システム

### Layer 1: 物理的上限フィルター

世界記録ベースの絶対上限を超えるデータは無条件で拒否。

| 区間 | 絶対値上限 | W/kg上限 | 元データ |
|---|---|---|---|
| 5秒 | 2500W | 25 W/kg | トラックスプリンター |
| 1分 | 1300W | 14 W/kg | ロード最高クラス |
| 5分 | 700W | 7.5 W/kg | グランツール上位 |
| 20分 | 530W | 6.4 W/kg | ボガチャル級 |
| 1時間 | 500W | 6.0 W/kg | アワーレコード |

**実装位置**: `/game/src/network/data-validator.js`

---

### Layer 2: 統計的異常値検出

過去データとの整合性チェック。

**検出ルール**:
1. **急激な実力向上**: 1ヶ月で20%以上のFTP向上 → 要レビュー
2. **ペアリングデータ不整合**: パワー > 0 かつ心拍 = 0 → 不審
3. **心拍とパワーの相関異常**: 期待値から30%以上乖離
4. **変動係数異常**: パワーが一定すぎる（CV < 2%）→ ボット疑い

---

### Layer 3: デバイスフィンガープリント

- ペアリングしたパワーメーターのBluetooth ID記録
- 同一デバイスからの複数アカウント検知
- BLE信号パターン解析
- センサー固有のノイズパターン記録

---

### Layer 4: コミュニティモデレーション

- 異常値検知時に「要レビュー」マーク
- リーダーボードから一時的に除外
- 他ユーザーからの通報機能

---

## ⚠️ チート判定時の対応階層

| Level | マーク | 対応 |
|---|---|---|
| 1 | 🟡 警告 | ユーザーに通知、ランキング反映保留 |
| 2 | 🟠 一時除外 | 該当セッション除外 |
| 3 | 🔴 アカウント凍結 | 機能制限、異議申し立て受付 |
| 4 | ⛔ 永久BAN | IP・デバイスブロック |

---

## 🏆 「Verified」バッジ制度

リーダーボードに信頼性のレイヤーを追加。

**Verified条件**:
- スマートトレーナー実機認証済み
- 心拍計同期確認済み
- 過去データ整合性チェック済み

---

## 🔐 全データ入力点での検証

### 入力範囲
- パワー: 0 ≤ P ≤ 2500W
- 心拍: 30 ≤ HR ≤ 220bpm
- 速度: 0 ≤ v ≤ 100km/h
- 体重: 30 ≤ m ≤ 200kg
- ケイデンス: 0 ≤ rpm ≤ 250

### 必須チェックポイント
- パワーメーター受信時
- ユーザープロフィール更新時
- セッション保存時
- リーダーボード送信時
- ゴーストデータ受信時

---

## 🔒 ユーザーデータ保護

- パスワードはbcryptハッシュ化
- 通信はTLS 1.3
- ローカルストレージは暗号化
- GDPR / 個人情報保護法準拠
