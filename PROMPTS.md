# 効果的だったプロンプト集

開発中に「これは効いた」というプロンプトを記録。

---

## 🔥 基本テンプレート

### コード生成時

```
以下の要件で実装してください。

【要件】
- ...

【制約】
- CLAUDE.mdの規約に準拠
- 1関数50行以内
- Vanilla JSのみ使用（ゲーム本体）
- ...

【出力形式】
- ファイル単位で完結
- 日本語コメント込み
- エラーハンドリング込み
```

### バグ修正時

```
以下のバグを修正してください。

【現象】
- ...

【期待する動作】
- ...

【再現手順】
1. ...

【関連ファイル】
- ...

修正前に、原因の仮説を3つ立ててから着手してください。
各仮説の検証方法も提示してください。
```

### 設計判断時

```
以下のテーマについて、ROLES.mdの役割で議論してください。

【テーマ】
- ...

【現状】
- ...

【選択肢】
- A: ...
- B: ...
- C: ...

各役割の意見と、最終的な推奨を出してください。
決定内容は DECISIONS.md に追記すべき形式で出力してください。
```

## 🎯 売上に貢献するプロンプト

### 価格設定の最適化

```
あなたは行動経済学の専門家です。
以下の価格戦略について、ユーザー心理から評価してください。

- Free / ¥980 / ¥1980 の3段階
- 年額34%オフ
- 7日間無料トライアル

特に「アンカリング効果」「松竹梅効果」「コミットメント効果」
の観点で改善案を出してください。
```

### 課金タイミング設計

```
あなたはF2Pゲームのマネタイズ専門家です。
ユーザーが課金したくなる「魔の瞬間」を5つ設計してください。

条件:
- ユーザー層: ライト/ヘビーで分けて
- 演出: ダーク過ぎず自然に
- 心理: 損失回避・希少性・社会的証明を活用
- ゲーム体験を損なわない範囲で
```

## 💡 隠れた名プロンプト

### ゼロから企画

```
あなたは元Zwiftのプロダクトマネージャーです。
もしZwiftの弱点を突いて新しいアプリを作るなら、
何を作りますか? 5つのアイデアを出してください。

各アイデアについて:
- ターゲットユーザー
- 差別化ポイント
- 想定収益モデル
- 実現難易度
を明記してください。
```

### 競合に厳しく分析

```
我々の新サービスについて、
ライバル企業の経営者になりきって厳しく批判してください。

- 想定される弱点
- 我々が見落としている可能性
- 失敗するシナリオ
を3つずつ挙げてください。
```

### ユーザーインタビュー風

```
35歳男性、サイクリスト3年目、年収700万円のペルソナを演じてください。

このユーザーがBITRIDERZについて初めて聞いた時の率直な感想と、
購入する/しない理由を述べてください。
```

### 失敗予測

```
我々のプロジェクトが1年後に失敗していたとします。
その理由を10個挙げてください。

各理由について:
- 発生確率
- 影響度
- 予防策
を明記してください。
```
